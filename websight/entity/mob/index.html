Mob

extends entity

Mobs are "living", moving game entities.

The term "mob" is short for "mobile".

  • Health

    float

    Amount of health the entity has.

  • AbsorptionAmount

    float

    Amount of extra health added by Absorption effect.

  • HurtTime

    short

    Number of ticks the mob turns red for after being hit. 0 when not recently hit.

  • HurtByTimestamp

    int

    The last time the mob was damaged, measured in the number of ticks since the mob's creation. Updates to a new value whenever the mob is damaged, then updates again 101 ticks later for reasons unknown. Can be changed with the entitydata command, but the specified value does not affect natural updates in any way, and is overwritten if the mob receives damage.

  • DeathTime

    short

    Number of ticks the mob has been dead for. Controls death animations. 0 when alive.

  • FallFlying

    byte

    Setting to 1 for non-player entities causes the entity to glide as long as they are wearing elytra in the chest slot. Can be used to detect when the player is gliding without using scoreboard statistics.

  • SleepingX

    int

    The X coordinate of where the entity is sleeping, absent if not sleeping.

  • SleepingY

    int

    The Y coordinate of where the entity is sleeping, absent if not sleeping.

  • SleepingZ

    int

    The Z coordinate of where the entity is sleeping, absent if not sleeping.

  • Brain

    compound

    Everything this entity has to keep in mind.

    • memories

      compound

      Empty for all but villagers.

      • minecraft:meeting_point

        compound

        Where this villager's meeting point is.

        • pos

          int [3]

          The X Y and Z values of the meeting point.

        • dimension

          string

          The dimension ID of the meeting point.

      • minecraft:home

        compound

        Where this villager's bed is.

        • pos

          int [3]

          The X Y and Z values of the bed.

        • dimension

          string

          The dimension ID of the bed.

      • minecraft:job_site

        compound

        Where this villager's job site block is.

        • pos

          int [3]

          The X Y and Z values of the job site block.

        • dimension

          string

          The dimension ID of the job site block.

  • Attributes

    attribute []

    A list of Attributes for this mob. These are used for many purposes in internal calculations, and can be considered a mob's "statistics". Valid Attributes for a given mob are listed in the main article.

  • ActiveEffects

    effect []

    The list of potion effects on this mob. May not exist.

  • HandItems

    Item [2]

    The list of items the mob is holding, in the following order: Main hand, Off hand.

  • ArmorItems

    Item [4]

    The list of items the mob is wearing as armor, in the following order: feet, legs, chest, head.

  • HandDropChances

    float [2]

    List of float values representing the chance for a handheld item to drop.

  • ArmorDropChances

    float [4]

    List of float values representing the chance for a worn armor item to drop.

  • DeathLootTable

    namespace id

    Optional. Loot table to be used for the items that drop when the entity is killed.

  • DeathLootTableSeed

    long

    Optional. Seed for generating the loot table. If 0 or omitted, a random seed is used.

  • CanPickUpLoot

    byte

    1 or 0 (true/false) - true if the mob can pick up loot (wear armor it picks up, use weapons it picks up).

  • NoAI

    byte

    1 or 0 (true/false) - Setting to true disables the mob's AI. The mob does not and cannot move, to the extent of not falling when normally able.

  • PersistenceRequired

    byte

    1 or 0 (true/false) - true if the mob must not despawn naturally.

  • LeftHanded

    byte

    1 or 0 (true/false) - true if the mob renders the main hand as being left.

  • Team

    string

    This tag is actually not part of the NBT data of a mob, but instead used when spawning it, so it cannot be tested for. It makes the mob instantly join the scoreboard team with that name.

  • Leashed

    byte

    1 or 0 (true/false) - whether the mob is leashed.

  • Leash

    compound

    Either contains a UUID long pair, if this leash connects to another entity\nOR\n an X, Y, Z int trio if this leash connects to a fencepost.

    • UUIDMost

      long

      The most significant bits of the Universally Unique IDentifier of the entity this leash connects to.

    • UUIDLeast

      long

      The least significant bits of the Universally Unique IDentifier of the entity this leash connects to.

    • X

      int

      The X coordinate of the fence this leash connects to.

    • Y

      int

      The Y coordinate of the fence this leash connects to.

    • Z

      int

      The Z coordinate of the fence this leash connects to.
